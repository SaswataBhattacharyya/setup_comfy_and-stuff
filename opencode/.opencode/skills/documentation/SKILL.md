## Goal

Improve project documentation.

## Workflow

1. Inspect existing documentation.
2. Identify missing setup, usage, architecture, or troubleshooting information.
3. Add concise, useful documentation.
4. Keep docs consistent with actual code and scripts.
5. Summarize what was added.

## Rules

- Do not document features that do not exist.
- Prefer examples and commands that actually match the repo.
